## chp 0.9 - build configs

_build configuration_ (aka build target) - collection of project settings that determines how your IDE build the project
_debug configuration_ - config designed to help debug program (offs optimizations, include debugging info etc.)
_release conifguration_ - designed to be used when releasing your program to public

## chp 0.10 - configuring compiler extension

_cpp standard_ - defines rules of how programs should behave (most cases compiler follows)
_compiler extensions_ - enables compiler specific behavior; many compiler implement their own changes to the language
    - often enabled by default; do not follow cpp standard
    - programs using non-standard extensions typically will not compile on other compilers
    - if they do, will not run correctly
    - best practice: DISABLE compiler extensions

## chp 0.11 - configuring compiler warning & error levels

- compiler checks to ensure you've followed cpp language rules
- programs that violate rules are _ill-formed_
- when issues are encountered, compiler emits _diagnostic message_
- cpp standard doesnt mandate a convention for diagnostic messages but compilers conventionally follow this:
    - diagnostic error / error - halt compilation (aka _compile errors_)
    - diagnostic warning / warning - compilation not halted, issue ignored
- diagnostic messages contain filename, line number & expected vs found

## chp 0.12 - choosing language standard

- many cpp standards available (cpp98, 03, 11, 14, 17, 20, 23 etc.)
- typically default is not the most recent; many default to cpp14 - missing latest features
- in professional envs, typically choose 1-2 versions back from latest finalized standard (i.e. if latest=20, use 17 or 14)
- for personal projects, use latest finalized standard

## chp 0.12 - what language standard is my compiler using?

use the program in `what_language_std.cpp` to print the name of the language standard
run it using:
- debug build: `g++ -o program what_language_std.cpp -O0 -g -Wall -Weffc++ -Wextra -Wconversion -Wsign-conversion -std=c++11`
- release build default cpp: `g++ -o program what_language_std.cpp -O2 -DNDEBUG -Wall -Weffc++ -Wextra -Wconversion -Wsign-conversion`
- release build cpp 14: `g++ -o program what_language_std.cpp -O2 -DNDEBUG -Wall -Weffc++ -Wextra -Wconversion -Wsign-conversion -std=c++14`
